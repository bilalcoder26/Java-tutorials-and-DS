4-->Can you explain briefly about the List interface?

> - Maintains insertion order (the order you add elements)

> - Allows null values (multiple nulls allowed)

**Common methods in List:**

| Method | What it does |
|--------|--------------|
| `get(index)` | Returns element at specific position |
| `set(index, element)` | Replaces element at position |
| `add(index, element)` | Inserts element at position |
| `remove(index)` | Removes element at position |
| `indexOf(element)` | Returns first position of element |
| `lastIndexOf(element)` | Returns last position of element |
| `subList(from, to)` | Returns portion of list |

> // Find index

**Common implementations of List:**

| Class | Description | When to use |
|-------|-------------|-------------|
| `ArrayList` | Resizable array | Most common, fast for get/set |
| `LinkedList` | Doubly linked list | Fast for add/remove at ends |

**Short version for interview:**
> "`List` is an ordered collection that allows duplicates and provides index-based access.

6--> How do you iterate around an ArrayList using iterator?
**Answer:**
> "We can iterate over an `ArrayList` using the `iterator()` method,
which returns an `Iterator` object. The iterator has `hasNext()` to check
if more elements exist and `next()` to get the next element."

**Example:**

> import java.util.ArrayList;
> import java.util.Iterator;
>
> public class IteratorExample {
>     public static void main(String[] args) {
>         ArrayList<String> fruits = new ArrayList<>();
>         fruits.add("Apple");
>         fruits.add("Banana");
>         fruits.add("Mango");
>         fruits.add("Orange");
>
>         // Get iterator
>         Iterator<String> iterator = fruits.iterator();
>
>         // Iterate using iterator
>         while (iterator.hasNext()) {
>             String fruit = iterator.next();
>             System.out.println(fruit);
>         }
>     }
> }

**Output:**
> ```
> Apple
> Banana
> Mango
> Orange
> ```

**Removing elements while iterating:**
> Iterator<String> iterator = fruits.iterator();
>
> while (iterator.hasNext()) {
>     String fruit = iterator.next();
>     if (fruit.equals("Banana")) {
>         iterator.remove();  // safe removal
>     }
> }
>
> System.out.println(fruits);  // [Apple, Mango, Orange]
> ```

**Important note (common interview trap):**
> // ❌ Wrong - Cannot modify list directly while iterating
> for (String fruit : fruits) {
>     if (fruit.equals("Banana")) {
>         fruits.remove(fruit);  // throws ConcurrentModificationException
>     }
> }
>
> // ✅ Correct - Use iterator's remove method
> Iterator<String> it = fruits.iterator();
> while (it.hasNext()) {
>     if (it.next().equals("Banana")) {
>         it.remove();  // safe
>     }
> }
> ```

**Other ways to iterate ArrayList (for comparison):**
> // 1. Simple for loop
> for (int i = 0; i < fruits.size(); i++) {
>     System.out.println(fruits.get(i));
> }
>
> // 2. Enhanced for loop
> for (String fruit : fruits) {
>     System.out.println(fruit);
> }
>
> // 3. forEach with lambda
> fruits.forEach(fruit -> System.out.println(fruit));
>
> // 4. Method reference
> fruits.forEach(System.out::println);
> ```

**Iterator methods summary:**

| Method | What it does |
|--------|--------------|
| `hasNext()` | Returns true if more elements exist |
| `next()` | Returns the next element |
| `remove()` | Removes the last element returned by `next()` |

**Short version for interview:**
> "Call `iterator()` on the ArrayList to get an Iterator.
Then use `while(iterator.hasNext())` loop and call `iterator.next()` to get each element.
To safely remove elements while iterating, use `iterator.remove()`, not the ArrayList's remove method."

7--> How do you sort an ArrayList?
**Answer:**
> "There are two common ways to sort an `ArrayList`: using `Collections.sort()`
for natural ordering, or using `list.sort()` with a custom comparator."

**Example 1: Natural order (numbers and strings)**

> import java.util.ArrayList;
> import java.util.Collections;
>
> ArrayList<Integer> numbers = new ArrayList<>();
> numbers.add(5);
> numbers.add(2);
> numbers.add(8);
> numbers.add(1);
>
> // Sort in ascending order (natural order)
> Collections.sort(numbers);
> System.out.println(numbers);  // [1, 2, 5, 8]
>
> // Sort strings alphabetically
> ArrayList<String> names = new ArrayList<>();
> names.add("Banana");
> names.add("Apple");
> names.add("Mango");
>
> Collections.sort(names);
> System.out.println(names);  // [Apple, Banana, Mango]
> ```

**Example 2: Reverse order**
> // Sort in reverse order
> Collections.sort(numbers, Collections.reverseOrder());
> System.out.println(numbers);  // [8, 5, 2, 1]

**Example 3: Using list.sort() method (Java 8+)**
> // Using list.sort() with Comparator
> numbers.sort(null);  // null means natural order
> System.out.println(numbers);  // [1, 2, 5, 8]
>
> // Custom comparator for reverse order
> numbers.sort((a, b) -> b - a);
> System.out.println(numbers);  // [8, 5, 2, 1]

**Example 4: Sorting custom objects**
> class Person {
>     String name;
>     int age;
>
>     Person(String name, int age) {
>         this.name = name;
>         this.age = age;
>     }
>
>     public String toString() {
>         return name + ":" + age;
>     }
> }
>
> ArrayList<Person> people = new ArrayList<>();
> people.add(new Person("John", 30));
> people.add(new Person("Alice", 25));
> people.add(new Person("Bob", 35));
>
> // Sort by age
> people.sort((p1, p2) -> p1.age - p2.age);
> System.out.println(people);  // [Alice:25, John:30, Bob:35]
>
> // Sort by name
> people.sort((p1, p2) -> p1.name.compareTo(p2.name));
> System.out.println(people);  // [Alice:25, Bob:35, John:30]

**Quick summary:**

| Method | Example |
|--------|---------|
| Natural order | `Collections.sort(list)` |
| Reverse order | `Collections.sort(list, Collections.reverseOrder())` |
| Custom order | `list.sort((a,b) -> a.compareTo(b))` |
| With Comparator | `Collections.sort(list, comparator)` |

**Short version for interview:**
> "Use `Collections.sort(arrayList)` for natural sorting.
For custom sorting, use `arrayList.sort((a,b) -> a.compareTo(b))` or pass a `Comparator`.
For reverse order, use `Collections.sort(list, Collections.reverseOrder())`."

8--> How do you sort elements in an ArrayList using comparable interface?
**Answer:**
> "To sort using `Comparable`, the class implements `Comparable<T>` and overrides the `compareTo()` method.
Then `Collections.sort(list)` will automatically use this natural ordering."

**Example:**
> import java.util.ArrayList;
> import java.util.Collections;
>
> // Step 1: Implement Comparable interface
> class Student implements Comparable<Student> {
>     int id;
>     String name;
>
>     Student(int id, String name) {
>         this.id = id;
>         this.name = name;
>     }
>
>     // Step 2: Override compareTo method
>     @Override
>     public int compareTo(Student other) {
>         // Sort by id (ascending)
>         return this.id - other.id;
>     }
>
>     public String toString() {
>         return id + ":" + name;
>     }
> }
>
> public class ComparableExample {
>     public static void main(String[] args) {
>         ArrayList<Student> students = new ArrayList<>();
>         students.add(new Student(103, "John"));
>         students.add(new Student(101, "Alice"));
>         students.add(new Student(102, "Bob"));
>
>         System.out.println("Before: " + students);  // [103:John, 101:Alice, 102:Bob]
>
>         // Step 3: Sort using Collections.sort()
>         Collections.sort(students);
>
>         System.out.println("After: " + students);   // [101:Alice, 102:Bob, 103:John]
>     }
> }

**compareTo() method contract:**

| Return value | Meaning |
|--------------|---------|
| Negative | this object is less than other |
| Zero | this object equals other |
| Positive | this object is greater than other |

**Example with String comparison:**
> ```java
> class Person implements Comparable<Person> {
>     String name;
>     int age;
>
>     Person(String name, int age) {
>         this.name = name;
>         this.age = age;
>     }
>
>     @Override
>     public int compareTo(Person other) {
>         // Sort by name alphabetically
>         return this.name.compareTo(other.name);
>     }
>
>     public String toString() {
>         return name + ":" + age;
>     }
> }
>
> // Usage
> ArrayList<Person> people = new ArrayList<>();
> people.add(new Person("John", 30));
> people.add(new Person("Alice", 25));
> people.add(new Person("Bob", 35));
>
> Collections.sort(people);
> System.out.println(people);  // [Alice:25, Bob:35, John:30]

**Example with multiple fields:**
> @Override
> public int compareTo(Employee other) {
>     // First compare by name
>     int nameCompare = this.name.compareTo(other.name);
>     if (nameCompare != 0) {
>         return nameCompare;
>     }
>     // If names are same, compare by id
>     return this.id - other.id;
> }

**Difference between Comparable and Comparator:**

| Feature | Comparable | Comparator |
|---------|------------|------------|
| Package | `java.lang` | `java.util` |
| Method | `compareTo(T o)` | `compare(T o1, T o2)` |
| Sorting | Natural ordering | Custom ordering |
| Changes class | Yes (modifies the class) | No (external) |

**Short version for interview:**
> "Implement `Comparable<T>` in the class and override `compareTo()`.
Define sorting logic inside `compareTo()`. Then call `Collections.sort(list)` to sort.
Example: `public int compareTo(Student s) { return this.id - s.id; }`"

9-->  How do you sort elements in an ArrayList using comparator interface?

**Answer:**
> "To sort using `Comparator`, we create a separate class that implements `Comparator<T>`
and overrides `compare()` method. Then pass an instance of this comparator to `Collections.sort(list, comparator)`."

**Example 1: Separate Comparator class**
> import java.util.*;
>
> // Class to be sorted
> class Student {
>     int id;
>     String name;
>
>     Student(int id, String name) {
>         this.id = id;
>         this.name = name;
>     }
>
>     public String toString() {
>         return id + ":" + name;
>     }
> }
>
> // Step 1: Create Comparator class
> class NameComparator implements Comparator<Student> {
>     @Override
>     public int compare(Student s1, Student s2) {
>         return s1.name.compareTo(s2.name);  // sort by name
>     }
> }
>
> class IdComparator implements Comparator<Student> {
>     @Override
>     public int compare(Student s1, Student s2) {
>         return s1.id - s2.id;  // sort by id
>     }
> }
>
> // Step 2: Use Comparator
> public class ComparatorExample {
>     public static void main(String[] args) {
>         ArrayList<Student> students = new ArrayList<>();
>         students.add(new Student(103, "John"));
>         students.add(new Student(101, "Alice"));
>         students.add(new Student(102, "Bob"));
>
>         System.out.println("Original: " + students);  // [103:John, 101:Alice, 102:Bob]
>
>         // Sort by name using NameComparator
>         Collections.sort(students, new NameComparator());
>         System.out.println("By name: " + students);   // [101:Alice, 102:Bob, 103:John]
>
>         // Sort by id using IdComparator
>         Collections.sort(students, new IdComparator());
>         System.out.println("By id: " + students);     // [101:Alice, 102:Bob, 103:John]
>     }
> }

**Example 2: Using anonymous class (shorter)**
> // Sort by name using anonymous comparator
> Collections.sort(students, new Comparator<Student>() {
>     @Override
>     public int compare(Student s1, Student s2) {
>         return s1.name.compareTo(s2.name);
>     }
> });
> ```

**Example 3: Using lambda expression (Java 8+ shortest)**
> // Sort by name
> students.sort((s1, s2) -> s1.name.compareTo(s2.name));
>
> // Sort by id
> students.sort((s1, s2) -> s1.id - s2.id);
>
> // Sort by name in reverse order
> students.sort((s1, s2) -> s2.name.compareTo(s1.name));
> ```

**Example 4: Multiple field sorting**
> ```java
> Comparator<Student> multiField = (s1, s2) -> {
>     // First sort by name
>     int nameCompare = s1.name.compareTo(s2.name);
>     if (nameCompare != 0) {
>         return nameCompare;
>     }
>     // If names are same, sort by id
>     return s1.id - s2.id;
> };
>
> students.sort(multiField);
> ```

**Example 5: Using Comparator helper methods (Java 8+)**
> ```java
> // Using Comparator.comparing()
> students.sort(Comparator.comparing(s -> s.name));
> students.sort(Comparator.comparing(s -> s.id));
>
> // Chaining comparators
> students.sort(Comparator.comparing(s -> s.name)
>                         .thenComparing(s -> s.id));
>
> // Reverse order
> students.sort(Comparator.comparing(s -> s.name).reversed());
> ```

**Summary of ways to use Comparator:**

| Method | Example |
|--------|---------|
| Separate class | `Collections.sort(list, new MyComparator())` |
| Anonymous class | `Collections.sort(list, new Comparator<T>(){...})` |
| Lambda | `list.sort((a,b) -> a.value - b.value)` |
| Helper methods | `list.sort(Comparator.comparing(T::getValue))` |

**Difference between Comparable and Comparator:**

| Feature | Comparable | Comparator |
|---------|------------|------------|
| Method | `compareTo()` | `compare()` |
| Changes original class | Yes | No |
| Multiple sorting options | No (only one) | Yes (many) |
| Usage | `Collections.sort(list)` | `Collections.sort(list, comparator)` |

**Short version for interview:**
> "Create a class implementing `Comparator<T>` and override `compare()`.
Pass it to `Collections.sort(list, comparator)`. Or use lambda: `list.sort((a,b) -> a.value - b.value)`.
Comparator allows multiple sorting options without modifying the original class."

10--> What is vector class? How is it different from an ArrayList?
**Answer:**
> "`Vector` is a legacy class (from Java 1.0) that implements a dynamic array similar to `ArrayList`.
The main difference is that `Vector` is **thread-safe** (synchronized), while `ArrayList` is not."

**Example:**
> import java.util.Vector;
> import java.util.ArrayList;
>
> public class VectorExample {
>     public static void main(String[] args) {
>         // Vector usage (similar to ArrayList)
>         Vector<String> vector = new Vector<>();
>         vector.add("Apple");
>         vector.add("Banana");
>         vector.add("Mango");
>
>         System.out.println(vector);        // [Apple, Banana, Mango]
>         System.out.println(vector.get(1)); // Banana
>
>         // Legacy methods (not in ArrayList)
>         System.out.println(vector.capacity());  // default capacity is 10
>     }
> }
> ```

**Key differences between Vector and ArrayList:**

| Feature | Vector | ArrayList |
|---------|--------|-----------|
| **Thread-safe** | Yes (synchronized) | No |
| **Performance** | Slower (due to synchronization) | Faster |
| **Introduced in** | Java 1.0 (legacy) | Java 1.2 |
| **Growth policy** | Doubles size (100% growth) | Grows by 50% |
| **Legacy methods** | Has extra methods like `elements()`, `capacity()` | No legacy methods |
| **When to use** | Rarely (use `CopyOnWriteArrayList` instead) | Most common |

**Performance example:**
> // Vector is synchronized - slower
> Vector<Integer> vector = new Vector<>();
> vector.add(10);  // synchronized method
>
> // ArrayList is not synchronized - faster
> ArrayList<Integer> list = new ArrayList<>();
> list.add(10);    // not synchronized

**Important note (common interview question):**
> "If you need thread-safe list, do **NOT** use `Vector` (it's legacy and has performance issues).
Instead use `Collections.synchronizedList(new ArrayList<>())` or `CopyOnWriteArrayList`."

**Modern alternatives for thread-safe lists:**
> // Option 1: Wrapped ArrayList
> List<String> syncList = Collections.synchronizedList(new ArrayList<>());
>
> // Option 2: CopyOnWriteArrayList (for read-heavy scenarios)
> List<String> copyList = new CopyOnWriteArrayList<>();

**Short version for interview:**
> "`Vector` is a thread-safe, synchronized version of `ArrayList`.
It is slower because of synchronization. `ArrayList` is not thread-safe but faster.
`Vector` is legacy and rarely used today — we prefer `ArrayList` with `Collections.synchronizedList()`
if thread-safety is needed."

11-->What is linkedList? What interfaces does it implement? How is it different from an ArrayList?
**Answer:**
> "`LinkedList` is a doubly-linked list implementation of `List` and `Deque` interfaces.
It stores elements as nodes where each node points to the previous and next node. It is different from `ArrayList`
 because it is better for insertions/deletions but worse for random access."

**Interfaces implemented by LinkedList:**
> // LinkedList implements these interfaces:
> // - List
> // - Deque (Double ended queue)
> // - Queue
> // - Cloneable
> // - Serializable
>
> LinkedList<String> list = new LinkedList<>();

**Example:**
> import java.util.LinkedList;
>
> public class LinkedListExample {
>     public static void main(String[] args) {
>         LinkedList<String> list = new LinkedList<>();
>
>         // List interface methods
>         list.add("Apple");
>         list.add("Banana");
>         list.add("Mango");
>         list.add(1, "Orange");  // insert at index
>
>         System.out.println(list);  // [Apple, Orange, Banana, Mango]
>
>         // Deque interface methods (additional)
>         list.addFirst("Grapes");   // add at beginning
>         list.addLast("Cherry");    // add at end
>
>         System.out.println(list);  // [Grapes, Apple, Orange, Banana, Mango, Cherry]
>
>         // Deque removal methods
>         String first = list.removeFirst();  // removes first
>         String last = list.removeLast();    // removes last
>
>         System.out.println(first);  // Grapes
>         System.out.println(last);   // Cherry
>         System.out.println(list);   // [Apple, Orange, Banana, Mango]
>
>         // Queue methods
>         list.offer("Pear");      // adds at end (queue operation)
>         String peek = list.peek();  // looks at first element
>         String poll = list.poll();  // removes and returns first
>     }
> }
> ```

**Key differences between LinkedList and ArrayList:**

| Feature | ArrayList | LinkedList |
|---------|-----------|------------|
| **Internal structure** | Dynamic array | Doubly linked list |
| **Get by index** | O(1) - very fast | O(n) - slow |
| **Add at end** | O(1) amortized | O(1) |
| **Add at middle** | O(n) - slow (shifting) | O(1) if you have reference, else O(n) |
| **Remove from middle** | O(n) - slow (shifting) | O(1) if you have reference, else O(n) |
| **Memory usage** | Less overhead (just array) | More (stores prev/next pointers) |
| **Implements Deque** | No | Yes |
| **Best for** | Frequent searching, random access | Frequent insertions/deletions, queue/stack operations |

**When to use which:**

> // Use ArrayList when:
> // - You need fast random access (get/set by index)
> // - You mostly iterate or search
> // - You rarely add/remove from middle
>
> // Use LinkedList when:
> // - You need frequent insertions/deletions at beginning or middle
> // - You need a Queue or Deque (FIFO, LIFO)
> // - You don't need fast random access

**Performance comparison example:**
> // ArrayList - fast get
> ArrayList<Integer> arrayList = new ArrayList<>();
> arrayList.add(10);
> System.out.println(arrayList.get(0));  // O(1) - very fast
>
> // LinkedList - slow get
> LinkedList<Integer> linkedList = new LinkedList<>();
> linkedList.add(10);
> System.out.println(linkedList.get(0)); // O(n) - must traverse
>
> // LinkedList - fast add at beginning
> linkedList.addFirst(5);   // O(1) - very fast
>
> // ArrayList - slow add at beginning (shifts all elements)
> arrayList.add(0, 5);      // O(n) - slow

**Short version for interview:**
> "`LinkedList` implements `List` and `Deque` interfaces.
It uses doubly-linked nodes. Unlike `ArrayList`, it has slower random access (O(n) vs O(1)) but
faster insertions/deletions at beginning/middle. It also supports queue operations like `addFirst()`,
`addLast()`, `poll()`, `peek()`."

15--> Explain briefly about Queue interface?

**Answer:**
> "The `Queue` interface is a collection designed for holding elements before processing.
It follows the **FIFO** (First-In-First-Out) principle, meaning elements are added at the end
and removed from the front."

**Key features:**
> - Follows FIFO order
> - Two sets of methods: one throws exception, one returns special value
> - Common implementations: `PriorityQueue`, `LinkedList`, `ArrayDeque`

**Queue methods (two styles):**

| Operation | Throws exception | Returns special value |
|-----------|-----------------|----------------------|
| Add | `add(e)` | `offer(e)` |
| Remove | `remove()` | `poll()` |
| Examine (peek) | `element()` | `peek()` |

**Example using LinkedList as Queue:**
> import java.util.*;
>
> public class QueueExample {
>     public static void main(String[] args) {
>         // Queue using LinkedList
>         Queue<String> queue = new LinkedList<>();
>
>         // Adding elements
>         queue.add("First");
>         queue.add("Second");
>         queue.add("Third");
>         queue.offer("Fourth");  // alternative add
>
>         System.out.println(queue);  // [First, Second, Third, Fourth]
>
>         // Examine front element
>         System.out.println(queue.element());  // First (throws if empty)
>         System.out.println(queue.peek());     // First (returns null if empty)
>
>         // Remove elements (FIFO order)
>         System.out.println(queue.remove());  // First
>         System.out.println(queue.poll());    // Second
>
>         System.out.println(queue);  // [Third, Fourth]
>     }
> }

**Output:**
> ```
> [First, Second, Third, Fourth]
> First
> First
> First
> Second
> [Third, Fourth]

**Common implementations:**

| Implementation | Description | Order |
|----------------|-------------|-------|
| `LinkedList` | Implements Queue as well as List | FIFO |
| `PriorityQueue` | Elements processed by priority, not FIFO | Based on priority |
| `ArrayDeque` | Resizable array, faster than LinkedList | FIFO or LIFO |

**PriorityQueue example (not FIFO):**
> Queue<Integer> pq = new PriorityQueue<>();
> pq.add(30);
> pq.add(10);
> pq.add(20);
>
> System.out.println(pq.poll());  // 10 (smallest first)
> System.out.println(pq.poll());  // 20
> System.out.println(pq.poll());  // 30

**Short version for interview:**
> "`Queue` follows FIFO (First-In-First-Out).
Key methods: `offer()` (add), `poll()` (remove), `peek()` (examine).
Common implementations: `LinkedList`, `PriorityQueue`, and `ArrayDeque`."

17--> Explain about the Deque interface?
**Answer:**
> "`Deque` stands for Double Ended Queue. It extends the `Queue` interface and
allows adding or removing elements from **both ends** (front and back).
It can be used as both a Queue (FIFO) and a Stack (LIFO)."

**Key features:**
> - Add/remove from both ends
> - Can be used as Queue (FIFO) or Stack (LIFO)
> - Methods for each operation in two styles (exception vs special value)

**Deque methods:**

| Operation | Throws exception (front) | Returns special (front) | Throws exception (back) | Returns special (back) |
|-----------|------------------------|----------------------|----------------------|---------------------|
| Add | `addFirst(e)` | `offerFirst(e)` | `addLast(e)` | `offerLast(e)` |
| Remove | `removeFirst()` | `pollFirst()` | `removeLast()` | `pollLast()` |
| Examine | `getFirst()` | `peekFirst()` | `getLast()` | `peekLast()` |

**Example:**
> import java.util.*;
>
> public class DequeExample {
>     public static void main(String[] args) {
>         // Using ArrayDeque (most common implementation)
>         Deque<String> deque = new ArrayDeque<>();
>
>         // Adding at both ends
>         deque.addFirst("First");
>         deque.addLast("Last");
>         deque.offerFirst("New First");
>         deque.offerLast("New Last");
>
>         System.out.println(deque);  // [New First, First, Last, New Last]
>
>         // Examine both ends
>         System.out.println(deque.getFirst());   // New First
>         System.out.println(deque.peekLast());   // New Last
>
>         // Remove from both ends
>         System.out.println(deque.removeFirst());  // New First
>         System.out.println(deque.pollLast());     // New Last
>
>         System.out.println(deque);  // [First, Last]
>     }
> }

**Using Deque as Queue (FIFO):**
> Deque<String> queue = new ArrayDeque<>();
>
> // Add at tail, remove from head
> queue.addLast("Task 1");
> queue.addLast("Task 2");
> queue.addLast("Task 3");
>
> while (!queue.isEmpty()) {
>     System.out.println(queue.removeFirst());  // Task 1, Task 2, Task 3
> }

**Using Deque as Stack (LIFO):**
> Deque<String> stack = new ArrayDeque<>();
>
> // Push and pop from same end
> stack.push("Item 1");
> stack.push("Item 2");
> stack.push("Item 3");
>
> System.out.println(stack.pop());  // Item 3
> System.out.println(stack.pop());  // Item 2
> System.out.println(stack.pop());  // Item 1

**Common implementations:**

| Implementation | Thread-safe | Performance | When to use |
|----------------|-------------|-------------|-------------|
| `ArrayDeque` | No | Fastest | Most common use |
| `LinkedList` | No | Good (more memory) | When need List operations too |
| `ConcurrentLinkedDeque` | Yes | Good | Multi-threaded environment |

**Difference between Queue and Deque:**

| Feature | Queue | Deque |
|---------|-------|-------|
| Add/remove from | Only one end (front for remove, back for add) | Both ends |
| Can be used as Stack | No | Yes |
| Methods | `offer()`, `poll()`, `peek()` | `addFirst/Last`, `removeFirst/Last`, etc. |

**Short version for interview:**
> "`Deque` is a Double Ended Queue that allows adding and removing from both ends.
It can be used as a Queue (FIFO) using `addLast()`/`removeFirst()` or as a Stack (LIFO) using `push()`/`pop()`.
Common implementation is `ArrayDeque`."

18--> Explain the BlockingQueue interface?
**Answer:**
> "`BlockingQueue` is a thread-safe queue interface that **blocks** a thread when trying to add
to a full queue or remove from an empty queue. It is designed for producer-consumer problems in
multi-threaded environments."

**Key features:**
> - Thread-safe (can be used by multiple threads)
> - Blocks when queue is full (on add) or empty (on remove)
> - Supports timeout options
> - Cannot add null elements

**BlockingQueue methods:**

| Operation | Throws exception | Special value | Blocks | Blocks with timeout |
|-----------|-----------------|---------------|--------|---------------------|
| Add | `add(e)` | `offer(e)` | `put(e)` | `offer(e, time, unit)` |
| Remove | `remove()` | `poll()` | `take()` | `poll(time, unit)` |
| Examine | `element()` | `peek()` | - | - |

**Complete example (Producer-Consumer):**
> import java.util.concurrent.*;
>
> class Producer implements Runnable {
>     private BlockingQueue<Integer> queue;
>
>     Producer(BlockingQueue<Integer> queue) {
>         this.queue = queue;
>     }
>
>     public void run() {
>         try {
>             for (int i = 1; i <= 5; i++) {
>                 queue.put(i);  // blocks if queue is full
>                 System.out.println("Produced: " + i);
>                 Thread.sleep(500);
>             }
>         } catch (InterruptedException e) {}
>     }
> }
>
> class Consumer implements Runnable {
>     private BlockingQueue<Integer> queue;
>
>     Consumer(BlockingQueue<Integer> queue) {
>         this.queue = queue;
>     }
>
>     public void run() {
>         try {
>             for (int i = 1; i <= 5; i++) {
>                 int value = queue.take();  // blocks if queue is empty
>                 System.out.println("Consumed: " + value);
>                 Thread.sleep(1000);
>             }
>         } catch (InterruptedException e) {}
>     }
> }
>
> public class BlockingQueueExample {
>     public static void main(String[] args) {
>         // Create a blocking queue with capacity 2
>         BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(2);
>
>         Thread producer = new Thread(new Producer(queue));
>         Thread consumer = new Thread(new Consumer(queue));
>
>         producer.start();
>         consumer.start();
>     }
> }

**Output example:**
> Produced: 1
> Produced: 2
> (producer blocks because queue is full)
> Consumed: 1
> Produced: 3
> Consumed: 2
> Produced: 4


**Common implementations:**

| Implementation | Description | Use case |
|----------------|-------------|----------|
| `ArrayBlockingQueue` | Bounded queue backed by array | Fixed capacity |
| `LinkedBlockingQueue` | Optionally bounded, backed by linked nodes | High throughput |
| `PriorityBlockingQueue` | Unbounded, elements by priority | Priority ordering |
| `SynchronousQueue` | Zero capacity (handoff only) | Direct handoff between threads |
| `DelayQueue` | Elements with delay | Scheduled tasks |

**Comparison with regular Queue:**

| Feature | Queue | BlockingQueue |
|---------|-------|---------------|
| Thread-safe | No (not by default) | Yes |
| Blocking on full | No | Yes (`put()` blocks) |
| Blocking on empty | No | Yes (`take()` blocks) |
| Null elements | Allowed | Not allowed |
| Use case | Single-threaded | Producer-consumer |

**Short version for interview:**
> "`BlockingQueue` is a thread-safe queue where `put()` blocks if queue is full and `take()`
blocks if queue is empty. It is used in producer-consumer problems. Common implementations:
`ArrayBlockingQueue` and `LinkedBlockingQueue`."

19--> What is a priorityQueue?
**Answer:**
> "`PriorityQueue` is a queue implementation where elements are processed
based on **priority** (natural order or custom comparator) rather than FIFO order.
The smallest or highest priority element is always at the front."

**Key features:**
> - Does **not** follow FIFO order
> - Elements are ordered by priority (natural order by default)
> - The head of the queue is the smallest (or highest priority) element
> - Does not allow null elements
> - Not thread-safe

**Example with natural order (numbers):**
> import java.util.PriorityQueue;
>
> public class PriorityQueueExample {
>     public static void main(String[] args) {
>         PriorityQueue<Integer> pq = new PriorityQueue<>();
>
>         // Add elements (not in order)
>         pq.add(30);
>         pq.add(10);
>         pq.add(50);
>         pq.add(20);
>         pq.add(40);
>
>         System.out.println("Queue: " + pq);  // [10, 20, 50, 30, 40] (heap order)
>
>         // Remove elements - comes out in sorted order (smallest first)
>         while (!pq.isEmpty()) {
>             System.out.print(pq.poll() + " ");  // 10 20 30 40 50
>         }
>     }
> }

**Example with strings (alphabetical order):**
> PriorityQueue<String> pq = new PriorityQueue<>();
> pq.add("Banana");
> pq.add("Apple");
> pq.add("Mango");
> pq.add("Cherry");
>
> while (!pq.isEmpty()) {
>     System.out.print(pq.poll() + " ");  // Apple Banana Cherry Mango
> }

**Example with custom comparator (reverse order):**
> // Max-heap (largest element first)
> PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
>
> maxHeap.add(30);
> maxHeap.add(10);
> maxHeap.add(50);
> maxHeap.add(20);
>
> while (!maxHeap.isEmpty()) {
>     System.out.print(maxHeap.poll() + " ");  // 50 30 20 10
> }

**Example with custom objects:**
> class Task implements Comparable<Task> {
>     String name;
>     int priority;
>
>     Task(String name, int priority) {
>         this.name = name;
>         this.priority = priority;
>     }
>
>     @Override
>     public int compareTo(Task other) {
>         return this.priority - other.priority;  // lower number = higher priority
>     }
>
>     public String toString() {
>         return name + "(" + priority + ")";
>     }
> }
>
> // Usage
> PriorityQueue<Task> taskQueue = new PriorityQueue<>();
> taskQueue.add(new Task("Email", 3));
> taskQueue.add(new Task("Report", 1));
> taskQueue.add(new Task("Backup", 2));
>
> while (!taskQueue.isEmpty()) {
>     System.out.print(taskQueue.poll() + " ");  // Report(1) Backup(2) Email(3)
> }

**Key methods:**

| Method | Description |
|--------|-------------|
| `add(e)` / `offer(e)` | Adds element |
| `poll()` | Removes and returns head (smallest element) |
| `peek()` | Returns head without removing |
| `remove()` | Removes head (throws if empty) |
| `size()` | Returns number of elements |

**Important notes:**
> - Does **not** maintain full sorted order internally (uses heap structure)
> - `poll()` and `peek()` always return the smallest element
> - Iteration does **not** guarantee sorted order
> - Time complexity: `offer()` and `poll()` are O(log n)

**PriorityQueue vs regular Queue:**

| Feature | Queue (LinkedList) | PriorityQueue |
|---------|-------------------|---------------|
| Order | FIFO (insertion order) | Based on priority (natural order) |
| Head element | First inserted element | Smallest (or highest priority) element |
| Use case | Simple task queue | Urgent tasks first, scheduling |

**Short version for interview:**
> "`PriorityQueue` orders elements by their natural order or a custom comparator.
The head is always the smallest (or highest priority) element. It does not follow FIFO.
Example: `PriorityQueue<Integer> pq = new PriorityQueue<>();` where `poll()` returns smallest element first."

22--> What is difference between Map and sortedMap?
**Answer:**
> "`Map` is the base interface that stores key-value pairs with no guarantee of order.
`SortedMap` extends `Map` and guarantees that keys are stored in **sorted order**
(either natural order or custom comparator order)."

**Key differences:**

| Feature | Map | SortedMap |
|---------|-----|-----------|
| **Order** | No guarantee of order | Keys are sorted |
| **Methods** | Basic: `put()`, `get()`, `remove()` | Extra: `firstKey()`, `lastKey()`, `headMap()`, `tailMap()`, `subMap()` |
| **Null keys** | Allowed in `HashMap` | Not allowed (throws NullPointerException) |
| **Implementations** | `HashMap`, `LinkedHashMap` | `TreeMap` |

**Example comparison:**

import java.util.*;

public class MapVsSortedMap {
    public static void main(String[] args) {
        // Map (HashMap) - no order
        Map<String, Integer> hashMap = new HashMap<>();
        hashMap.put("Banana", 3);
        hashMap.put("Apple", 1);
        hashMap.put("Mango", 2);
        System.out.println("HashMap: " + hashMap);  // {Apple=1, Banana=3, Mango=2} (no order)

        // SortedMap (TreeMap) - sorted by keys
        SortedMap<String, Integer> treeMap = new TreeMap<>();
        treeMap.put("Banana", 3);
        treeMap.put("Apple", 1);
        treeMap.put("Mango", 2);
        System.out.println("TreeMap: " + treeMap);  // {Apple=1, Banana=3, Mango=2} (sorted)

        // SortedMap extra methods
        System.out.println("First key: " + treeMap.firstKey());  // Apple
        System.out.println("Last key: " + treeMap.lastKey());    // Mango

        // Head map (keys less than "Banana")
        System.out.println("HeadMap: " + treeMap.headMap("Banana"));  // {Apple=1}

        // Tail map (keys greater than or equal to "Banana")
        System.out.println("TailMap: " + treeMap.tailMap("Banana"));  // {Banana=3, Mango=2}

        // Sub map (keys between "Apple" and "Mango")
        System.out.println("SubMap: " + treeMap.subMap("Apple", "Mango"));  // {Apple=1, Banana=3}
    }
}

**SortedMap extra methods:**

| Method | Description |
|--------|-------------|
| `firstKey()` | Returns the smallest key |
| `lastKey()` | Returns the largest key |
| `headMap(toKey)` | Returns view of keys less than toKey |
| `tailMap(fromKey)` | Returns view of keys greater than or equal to fromKey |
| `subMap(fromKey, toKey)` | Returns view of keys between fromKey and toKey |

**Important note (null handling):**

// Map allows null key
Map<String, String> hashMap = new HashMap<>();
hashMap.put(null, "value");  // ✅ allowed

// SortedMap does NOT allow null key
SortedMap<String, String> treeMap = new TreeMap<>();
treeMap.put(null, "value");  // ❌ NullPointerException

**Hierarchy:**

Map
 └── SortedMap
       └── NavigableMap (extends SortedMap)
             └── TreeMap

**Short version for interview:**
> "`Map` has no order guarantee. `SortedMap` extends `Map` and keeps keys in sorted order.
`SortedMap` provides extra methods like `firstKey()`, `lastKey()`, `headMap()`, and `tailMap()`.
`TreeMap` is the main implementation of `SortedMap`."

22--> What is a HashMap?

**Answer:**
> "`HashMap` is a hash table-based implementation of the `Map` interface.
It stores key-value pairs and provides **O(1)** average time complexity for `put()`, `get()`, and `remove()`
operations. It does **not guarantee any order** of keys."

**Key features:**
> - Allows one `null` key and multiple `null` values
> - No order guarantee (order can change over time)
> - Not thread-safe (use `ConcurrentHashMap` for multi-threading)
> - Fast performance (O(1) average for basic operations)

**Example:**

import java.util.*;

public class HashMapExample {
    public static void main(String[] args) {
        // Create a HashMap
        HashMap<String, Integer> map = new HashMap<>();

        // Add key-value pairs
        map.put("Apple", 10);
        map.put("Banana", 20);
        map.put("Mango", 30);
        map.put("Apple", 40);  // overwrites previous value for key "Apple"
        map.put(null, 50);     // one null key allowed
        map.put("Orange", null); // null value allowed

        System.out.println(map);  // {null=50, Apple=40, Banana=20, Orange=null, Mango=30}

        // Get value by key
        System.out.println(map.get("Apple"));     // 40
        System.out.println(map.get("Grapes"));    // null (key not found)

        // Check existence
        System.out.println(map.containsKey("Banana"));  // true
        System.out.println(map.containsValue(20));      // true

        // Remove entry
        map.remove("Mango");
        System.out.println(map);  // {null=50, Apple=40, Banana=20, Orange=null}

        // Get size
        System.out.println("Size: " + map.size());  // 4

        // Iterate over keys
        for (String key : map.keySet()) {
            System.out.println(key + " = " + map.get(key));
        }

        // Iterate over entries
        for (Map.Entry<String, Integer> entry : map.entrySet()) {
            System.out.println(entry.getKey() + " : " + entry.getValue());
        }

        // Iterate over values
        for (Integer value : map.values()) {
            System.out.println(value);
        }

        // Check if empty
        System.out.println(map.isEmpty());  // false

        // Clear all
        map.clear();
        System.out.println(map.isEmpty());  // true
    }
}
```

**Common methods:**

| Method | Description |
|--------|-------------|
| `put(key, value)` | Adds or updates key-value pair |
| `get(key)` | Returns value for key (or null) |
| `remove(key)` | Removes entry for key |
| `containsKey(key)` | Checks if key exists |
| `containsValue(value)` | Checks if value exists |
| `keySet()` | Returns Set of all keys |
| `values()` | Returns Collection of all values |
| `entrySet()` | Returns Set of all entries |
| `size()` | Returns number of entries |
| `isEmpty()` | Checks if map is empty |
| `clear()` | Removes all entries |

**HashMap internal working (brief):**

> - Uses array of `Node` (buckets)
> - `hashCode()` determines which bucket key goes to
> - `equals()` checks for key equality
> - Collisions handled by linked list or tree (Java 8+)

**Performance (Big O):**

| Operation | Average | Worst case |
|-----------|---------|------------|
| `put()` | O(1) | O(log n) or O(n) |
| `get()` | O(1) | O(log n) or O(n) |
| `remove()` | O(1) | O(log n) or O(n) |

**Initial capacity and load factor:**

// Default: capacity = 16, load factor = 0.75
HashMap<String, Integer> map1 = new HashMap<>();

// Custom capacity and load factor
HashMap<String, Integer> map2 = new HashMap<>(32, 0.8f);

**HashMap vs other Map implementations:**

| Feature | HashMap | LinkedHashMap | TreeMap |
|---------|---------|---------------|---------|
| Order | No order | Insertion order | Sorted order |
| Performance | O(1) | O(1) | O(log n) |
| Null key | Yes (one) | Yes (one) | No |
| When to use | Default choice | Need insertion order | Need sorted keys |

**Short version for interview:**
> "`HashMap` stores key-value pairs with O(1) average performance for put and get.
It allows one null key and multiple null values. It does not guarantee any order.
It is not thread-safe. Use `HashMap` when you don't care about order and need fast access."

23-->What is a TreeMap? How is different from a HashMap?
**Answer:**
> "`TreeMap` is a `SortedMap` implementation that stores key-value pairs in **sorted order**
based on the natural order of keys or a custom comparator.
It is different from `HashMap` because it maintains order but has slower O(log n) performance."

**Example:**

import java.util.*;

public class TreeMapExample {
    public static void main(String[] args) {
        // Create a TreeMap (sorted by keys)
        TreeMap<String, Integer> treeMap = new TreeMap<>();

        // Add elements (order doesn't matter)
        treeMap.put("Banana", 3);
        treeMap.put("Apple", 1);
        treeMap.put("Mango", 2);
        treeMap.put("Cherry", 4);

        // Keys are automatically sorted
        System.out.println(treeMap);  // {Apple=1, Banana=3, Cherry=4, Mango=2}

        // Get first and last keys
        System.out.println("First key: " + treeMap.firstKey());  // Apple
        System.out.println("Last key: " + treeMap.lastKey());    // Mango

        // Navigation methods
        System.out.println("Lower key than Banana: " + treeMap.lowerKey("Banana"));  // Apple
        System.out.println("Higher key than Banana: " + treeMap.higherKey("Banana")); // Cherry

        // Range views
        System.out.println("HeadMap (less than Banana): " + treeMap.headMap("Banana"));  // {Apple=1}
        System.out.println("TailMap (>= Banana): " + treeMap.tailMap("Banana"));  // {Banana=3, Cherry=4, Mango=2}
        System.out.println("SubMap (Apple to Mango): " + treeMap.subMap("Apple", "Mango"));  // {Apple=1, Banana=3, Cherry=4}

        // Remove
        treeMap.remove("Cherry");
        System.out.println(treeMap);  // {Apple=1, Banana=3, Mango=2}
    }
}
```

**Example with custom comparator (reverse order):**

// TreeMap with reverse order
TreeMap<String, Integer> reverseMap = new TreeMap<>(Comparator.reverseOrder());

reverseMap.put("Banana", 3);
reverseMap.put("Apple", 1);
reverseMap.put("Mango", 2);

System.out.println(reverseMap);  // {Mango=2, Banana=3, Apple=1}
```

**Key differences between TreeMap and HashMap:**

| Feature | HashMap | TreeMap |
|---------|---------|---------|
| **Order** | No order guarantee | Sorted order (natural/comparator) |
| **Performance** | O(1) average | O(log n) |
| **Null key** | Yes (one null) | No (throws NullPointerException) |
| **Null values** | Yes | Yes |
| **Thread-safe** | No | No |
| **Memory** | Less overhead | More overhead (tree structure) |
| **Navigation methods** | No | Yes (firstKey, lastKey, headMap, etc.) |
| **When to use** | Default choice, fast access | Need sorted keys or range queries |

**Performance comparison:**

// HashMap - faster for most operations
HashMap<String, Integer> hashMap = new HashMap<>();
hashMap.put("A", 1);  // O(1)
int val = hashMap.get("A");  // O(1)

// TreeMap - slower but sorted
TreeMap<String, Integer> treeMap = new TreeMap<>();
treeMap.put("A", 1);  // O(log n)
int val2 = treeMap.get("A");  // O(log n)
```

**Null key handling:**

// HashMap allows one null key
HashMap<String, Integer> hashMap = new HashMap<>();
hashMap.put(null, 100);  // ✅ allowed

// TreeMap does NOT allow null key
TreeMap<String, Integer> treeMap = new TreeMap<>();
treeMap.put(null, 100);  // ❌ NullPointerException
```

**When to use which:**

| Use HashMap when | Use TreeMap when |
|-----------------|------------------|
| Order doesn't matter | Keys need to be sorted |
| Fast performance needed | Need range queries (headMap, tailMap) |
| Need null key | Need navigation methods (lowerKey, higherKey) |
| Most common use case | Sorted output or dictionary-like data |

**SortedMap/NavigableMap methods in TreeMap:**

| Method | Description |
|--------|-------------|
| `firstKey()` | Smallest key |
| `lastKey()` | Largest key |
| `lowerKey(key)` | Greatest key less than given key |
| `higherKey(key)` | Smallest key greater than given key |
| `headMap(toKey)` | Keys less than toKey |
| `tailMap(fromKey)` | Keys greater than or equal to fromKey |
| `subMap(from, to)` | Keys between from and to |
| `pollFirstEntry()` | Removes and returns first entry |
| `pollLastEntry()` | Removes and returns last entry |

**Short version for interview:**
> "`TreeMap` stores keys in sorted order (O(log n) performance).
It does not allow null keys. It provides navigation methods like `firstKey()`, `lastKey()`, `headMap()`,
and `tailMap()`. Use `HashMap` for faster O(1) access when order doesn't matter; use `TreeMap`
when you need sorted keys or range queries."

24--> Can you give an example of implementation of navigableMap interface?

**Answer:**
> "`TreeMap` is the main implementation of the `NavigableMap` interface.
 It provides navigation methods to find closest matches, reverse order, and get subsets of the map."

**Example:**

import java.util.*;

public class NavigableMapExample {
    public static void main(String[] args) {
        // TreeMap implements NavigableMap
        NavigableMap<Integer, String> map = new TreeMap<>();

        // Add entries
        map.put(10, "Ten");
        map.put(20, "Twenty");
        map.put(30, "Thirty");
        map.put(40, "Forty");
        map.put(50, "Fifty");

        System.out.println("Original: " + map);  // {10=Ten, 20=Twenty, 30=Thirty, 40=Forty, 50=Fifty}

        // Closest matches
        System.out.println("Lower (less than 25): " + map.lowerKey(25));     // 20
        System.out.println("Higher (greater than 25): " + map.higherKey(25)); // 30
        System.out.println("Floor (≤ 25): " + map.floorKey(25));     // 20
        System.out.println("Ceiling (≥ 25): " + map.ceilingKey(25)); // 30

        // First and last
        System.out.println("First key: " + map.firstKey());  // 10
        System.out.println("Last key: " + map.lastKey());    // 50

        // Reverse order
        System.out.println("Descending map: " + map.descendingMap());  // {50=Fifty, 40=Forty, 30=Thirty, 20=Twenty, 10=Ten}

        // Remove first and last
        System.out.println("Poll first: " + map.pollFirstEntry());  // 10=Ten
        System.out.println("Poll last: " + map.pollLastEntry());    // 50=Fifty
        System.out.println("After poll: " + map);  // {20=Twenty, 30=Thirty, 40=Forty}

        // Subset views
        System.out.println("HeadMap (keys < 30): " + map.headMap(30));     // {20=Twenty}
        System.out.println("TailMap (keys ≥ 30): " + map.tailMap(30));     // {30=Thirty, 40=Forty}
        System.out.println("SubMap (20 to 40): " + map.subMap(20, 40));    // {20=Twenty, 30=Thirty}
    }
}

**NavigableMap methods summary:**

| Method | Description |
|--------|-------------|
| `lowerKey(key)` | Greatest key strictly less than given key |
| `higherKey(key)` | Smallest key strictly greater than given key |
| `floorKey(key)` | Greatest key less than or equal to given key |
| `ceilingKey(key)` | Smallest key greater than or equal to given key |
| `firstKey()` / `lastKey()` | First and last key |
| `pollFirstEntry()` / `pollLastEntry()` | Remove and return first/last entry |
| `descendingMap()` | Returns map in reverse order |
| `headMap(toKey)` | Keys less than toKey |
| `tailMap(fromKey)` | Keys greater than or equal to fromKey |
| `subMap(from, to)` | Keys between from and to |

**Short version for interview:**
> "`TreeMap` implements `NavigableMap`. It provides navigation methods like `lowerKey()`,
`higherKey()`, `floorKey()`, `ceilingKey()`, `pollFirstEntry()`, `pollLastEntry()`, and `descendingMap()`."

25--> What are the static methods present in the collections class?


**Answer:**
> "The `Collections` class provides many static utility methods for working with collections.
Common ones include `sort()`, `reverse()`, `shuffle()`, `binarySearch()`, `max()`, `min()`, `synchronizedXXX()`,
and `unmodifiableXXX()`."

**Examples of important static methods:**

| Method | Description |
|--------|-------------|
| `sort(list)` | Sorts list in natural order |
| `sort(list, comparator)` | Sorts list with custom comparator |
| `reverse(list)` | Reverses order of list |
| `shuffle(list)` | Randomly shuffles elements |
| `binarySearch(list, key)` | Searches sorted list (returns index) |
| `max(collection)` | Returns maximum element |
| `min(collection)` | Returns minimum element |
| `frequency(collection, element)` | Counts occurrences |
| `swap(list, i, j)` | Swaps elements at positions |
| `fill(list, obj)` | Replaces all elements with same object |
| `copy(dest, src)` | Copies from source to destination |
| `replaceAll(list, old, new)` | Replaces all occurrences |
| `rotate(list, distance)` | Rotates elements |
| `synchronizedList(list)` | Returns thread-safe list |
| `synchronizedSet(set)` | Returns thread-safe set |
| `synchronizedMap(map)` | Returns thread-safe map |
| `unmodifiableList(list)` | Returns read-only list |
| `unmodifiableSet(set)` | Returns read-only set |
| `unmodifiableMap(map)` | Returns read-only map |
| `emptyList()` | Returns empty immutable list |
| `emptySet()` | Returns empty immutable set |
| `emptyMap()` | Returns empty immutable map |
| `singletonList(obj)` | Returns immutable list with one element |
| `addAll(collection, elements...)` | Adds multiple elements |

**Complete example:**

import java.util.*;

public class CollectionsClassExample {
    public static void main(String[] args) {
        // Sorting
        List<Integer> list = new ArrayList<>(Arrays.asList(5, 2, 8, 1, 9));
        Collections.sort(list);
        System.out.println("Sorted: " + list);  // [1, 2, 5, 8, 9]

        // Reverse
        Collections.reverse(list);
        System.out.println("Reversed: " + list);  // [9, 8, 5, 2, 1]

        // Shuffle
        Collections.shuffle(list);
        System.out.println("Shuffled: " + list);  // random order

        // Binary search (list must be sorted first)
        Collections.sort(list);
        int index = Collections.binarySearch(list, 5);
        System.out.println("Index of 5: " + index);

        // Max and Min
        System.out.println("Max: " + Collections.max(list));  // 9
        System.out.println("Min: " + Collections.min(list));  // 1

        // Frequency
        List<String> names = Arrays.asList("A", "B", "A", "C", "A");
        System.out.println("Frequency of A: " + Collections.frequency(names, "A"));  // 3

        // Swap
        Collections.swap(list, 0, 4);
        System.out.println("After swap: " + list);

        // Fill
        Collections.fill(list, 0);
        System.out.println("After fill: " + list);  // [0, 0, 0, 0, 0]

        // Replace all
        List<String> words = new ArrayList<>(Arrays.asList("cat", "dog", "cat", "bird"));
        Collections.replaceAll(words, "cat", "animal");
        System.out.println("After replace: " + words);  // [animal, dog, animal, bird]

        // Thread-safe wrappers
        List<Integer> syncList = Collections.synchronizedList(new ArrayList<>());
        Set<String> syncSet = Collections.synchronizedSet(new HashSet<>());
        Map<String, Integer> syncMap = Collections.synchronizedMap(new HashMap<>());

        // Unmodifiable (read-only) wrappers
        List<Integer> readOnlyList = Collections.unmodifiableList(list);
        // readOnlyList.add(10);  // throws UnsupportedOperationException

        // Empty collections
        List<String> emptyList = Collections.emptyList();
        Set<String> emptySet = Collections.emptySet();
        Map<String, String> emptyMap = Collections.emptyMap();

        // Singleton collections
        List<String> singleList = Collections.singletonList("OnlyOne");
        Set<String> singleSet = Collections.singleton("OnlyOne");
        Map<String, Integer> singleMap = Collections.singletonMap("Key", 100);

        // Add all elements
        List<Integer> numbers = new ArrayList<>();
        Collections.addAll(numbers, 10, 20, 30, 40, 50);
        System.out.println("AddAll: " + numbers);  // [10, 20, 30, 40, 50]

        // Rotate
        Collections.rotate(numbers, 2);
        System.out.println("After rotate: " + numbers);  // [40, 50, 10, 20, 30]
    }
}
