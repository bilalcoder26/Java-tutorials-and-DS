# Reflection in Java — Complete Detailed Guide

Reflection in Java is a feature that allows a program to:

* Inspect classes
* Access fields
* Call methods
* Create objects dynamically at runtime

Even if the details are not known during compile time.

Reflection is part of:

```java
java.lang.reflect
```

---

# 1. What is Reflection?

Normally in Java:

```java
Student s = new Student();
s.show();
```

Everything is known at compile time.

With Reflection:

* Class can be loaded dynamically
* Methods can be called dynamically
* Fields can be modified dynamically

---

# 2. Why Reflection is Used?

Reflection is heavily used in:

* Spring Framework
* Hibernate
* JUnit
* Dependency Injection
* Annotations processing
* IDE tools
* Serialization
* Debugging tools

# 6. Ways to Get Class Object

---

## Method 1 — `.class`

```java
Class<?> c = Student.class;
```

---

```java
Student s = new Student();
```

# 7. Example Class

```java
class Student {

    private String name;

    public Student() {}

    public Student(String name) {
        this.name = name;
    }

    public void show() {
        System.out.println("Hello " + name);
    }

    private void secret() {
        System.out.println("Private Method");
    }
}
```

# 8. Getting Class Information

```java
Class<?> c = Student.class;

System.out.println(c.getName());
System.out.println(c.getSimpleName());
```

Output:

```text
Student
Student
```

# 9. Getting Constructors

```java
Constructor<?>[] constructors = c.getConstructors();

for(Constructor<?> con : constructors) {
    System.out.println(con);
}
```

---

## Get Specific Constructor

```java
Constructor<?> con = c.getConstructor(String.class);
```

# 10. Creating Objects Dynamically

```java
Constructor<?> con = c.getConstructor(String.class);

Object obj = con.newInstance("Bilal");
```

Equivalent to:

```java
new Student("Bilal");
```

---

# 11. Getting Methods

```java
Method[] methods = c.getMethods();

for(Method m : methods) {
    System.out.println(m.getName());
}
```

---

## Get Specific Method

```java
Method method = c.getMethod("show");
```

---

# 12. Invoking Methods Dynamically

```java
method.invoke(obj);
```

Equivalent to:

```java
obj.show();
```

# 13. Accessing Private Methods

Normally:

```java
private void secret()
```

Cannot be accessed outside class.

Reflection can bypass this.

---

## Example

```java
Method m = c.getDeclaredMethod("secret");

m.setAccessible(true);

m.invoke(obj);
```

Output:

```text
Private Method
```

---

# 14. Accessing Fields

```java
Field[] fields = c.getDeclaredFields();

for(Field f : fields) {
    System.out.println(f.getName());
}
```

---

# 15. Accessing Private Fields

```java
Field field = c.getDeclaredField("name");

field.setAccessible(true);

field.set(obj, "Ahmed");

System.out.println(field.get(obj));
```

# 16. Complete Reflection Example

```java
import java.lang.reflect.*;

class Student {

    private String name;

    public Student(String name) {
        this.name = name;
    }

    public void show() {
        System.out.println("Hello " + name);
    }

    private void secret() {
        System.out.println("Secret Method");
    }
}

public class Main {

    public static void main(String[] args) throws Exception {

        Class<?> c = Student.class;

        Constructor<?> con = c.getConstructor(String.class);

        Object obj = con.newInstance("Bilal");

        Method method = c.getMethod("show");

        method.invoke(obj);

        Method privateMethod = c.getDeclaredMethod("secret");

        privateMethod.setAccessible(true);

        privateMethod.invoke(obj);
    }
}
```

Output:

```text
Hello Bilal
Secret Method
```

# 20. Reflection and Encapsulation

Reflection can break encapsulation.

Example:

```java
private field
private method
```

Can still be accessed.

That's why reflection should be used carefully.

---

# 21. Performance Issue

Reflection is slower because:

* JVM performs dynamic resolution
* Security checks happen
* Optimization is harder

Direct method calls are much faster.

---

# 23. Difference Between `getMethods()` and `getDeclaredMethods()`

---

## `getMethods()`

Returns:

* Public methods
* Including inherited methods

---

## `getDeclaredMethods()`

Returns:

* All methods of current class
* Including private methods
* Excludes inherited methods

# 29. Tricky Interview Questions

---

## Q1

```java
Class<?> c1 = String.class;
Class<?> c2 = "Hello".getClass();

System.out.println(c1 == c2);
```

Output:

```text
true
```

Because both represent same class object.

---

## Q2

```java
privateMethod.setAccessible(true);
```

What does it do?

It bypasses Java access control checks.

---

## Q3

```java
Method m = c.getMethod("show");
```

Will this fetch private methods?

❌ No

Only public methods.

Use:

```java
getDeclaredMethod()
```

---

## Q4

```java
Field f = c.getField("name");
```

Will it access private field?

❌ No

Use:

```java
getDeclaredField()
```
